# Hanlog — 韓国語学習アプリ

> **v1 MVP 完成・v2 開発中。** デプロイ URL は準備中です（公開予定ドメイン: `hanlog.app`）

---

## 概要

韓国語学習を「インプット → 単語登録 → フラッシュカード学習 → 日記アウトプット」の1サイクルで完結できるようにした Web アプリです。
フロントエンド（React）とバックエンド（Rails API）を分離したモノレポ構成で、v1 の MVP をベースに v2 として公開サービス化を進めています。

---

## 画面の説明

### ログイン・新規登録（v2 で刷新予定）

メールアドレス＋パスワードによる認証画面です。v2 ではメール認証・パスワードリセット・リフレッシュトークンを追加予定です。

ログイン

### 単語帳

登録済みの単語を一覧管理する画面です。Unsplash API と連携しており、単語に合った画像を検索して添付できます。習熟度（0〜100）がバーで表示され、フラッシュカードの結果が自動反映されます。（習熟度をもとにしてSpaced Repetition（間隔反復）を組み込んだ効率的な学習システムを構想中）

単語帳

### 単語追加

韓国語・日本語訳・難易度と、Unsplash から検索した画像を一緒に登録できます。画像を単語学習に組み込むことで効率的な学習が期待できます。（①Dual Coding Theory「人は情報を言語情報と視覚情報の2経路で記憶できるという考え方」②Picture Superiority Effect「画像は文字より記憶に残りやすいという現象」）

単語追加

### フラッシュカード（表・裏）

単語帳からランダムに出題されます。「知っている / 知らない」で答えると習熟度が自動更新されます（正解 +10 / 不正解 -10）
（今後は忘却曲線に基づく自動更新機能を組み込みたいと思っています）


| 表（韓国語）    | 裏（日本語訳）   |
| --------- | --------- |
| フラッシュカード表 | フラッシュカード裏 |

## 技術的なこだわり

**① AI 抽象化レイヤの設計（実装済み）**
AI サービスは将来変わる可能性があるので、呼び出し口を1つにまとめた `AI::Client` を作りました。今はコスト面の考慮から Gemini を利用する前提ですが、将来他のAIモデルに切り替えたくなっても、アプリ側のコードを変えなくて済む構造にしています。テスト時に本物の API を呼ばないスタブ（`NullClient`）も用意しており、余計なコストや不安定さを出さないようにしています。

**② JWT + Refresh Token 認証（v2 実装予定）**
v1 では JWT のみですが、v2 では Access Token（短命）と Refresh Token（長命）に分ける設計を学んで取り入れようとしています。Access Token の有効期限を短くすることで、万が一漏れたときの影響を小さくできるためです。Refresh Token は JavaScript から読めない場所（HttpOnly Cookie）に置く予定です。

**③ Service Object による責務分離（v2 実装予定）**
学習中に「ロジックがコントローラーに集まりすぎると管理しにくくなる（Fat Controller）」という問題を知り、複雑な処理は `app/services/` に分けて書くようにしようと思っています。AI 校閲や習熟度の更新などを1つのクラスに1つの役割で書くことで、テストもしやすくなると考えたからです。

**④ TDD + CI（v2 から取り組んでいる実装手法）**
テストを先に書いてから実装する TDD に取り組んでいます。GitHub Actions で RuboCop・RSpec・Brakeman（セキュリティチェック）が push のたびに自動で走るCI環境を作りました。CI をきちんと整え、コードレビューは CodeRabbit（AI）を利用して効率的に行い、自分は「設計の意図と合っているか」の確認に集中できるようにしています。

## 背景

自分自身が韓国語を勉強していて、伸び悩んでいたりモチベーションが続きにくいという問題があったので「既存サービスの欲しい所を集めて自分が使いやすいツールを自分で作ろう」という動機でこのアプリを作り始めました。
AI を使って自分が書いた日記の文法や表現をチェック・校閲してもらったり、記事から単語を自動で抽出してもらうことで、「インプット → 単語登録 → フラッシュカード学習 → 日記アウトプット」のサイクルを作り効率的な学習が１つのサービスで回せすことが目的のひとつです。

このアプリはそれ自体が目的であると同時に、**より大きな構想への入り口**でもあります。
韓国語学習者を最初のユーザーとして集め、このアプリで形を作った後に、英語・料理・筋トレなど他のジャンルにも展開できる「大人向け学習プラットフォーム」に発展させていく計画を持っています。

---

## 解決したい課題

既存の学習アプリの多くは「ユーザーが自発的に続けること」を前提に設計されているため、止まったときにプレッシャーや罪悪感が生まれやすかったり、そもそもモチベーションの動機がアプリに依存しない構造になりがちです。
このアプリ（そして将来のプラットフォーム）が解決したいのは、**「続けたいのに続けられない」という状態**です。
ユーザーが【やらされている感なく】「やりたい！」と思わせることができるような設計を心掛けています

- 成果が見えやすい、成長が実感しやすい設計にすることを１番意識している。これによりユーザーが続けやすくなる
- 一度やめてもいつでも再起動しやすい動線、設計
- 学習の成果だけでなく、寄り添う AI フィードバック
- 「次に何をすればいいか」を AI が文脈の中で自然に提案することで学習者の迷走を防ぐ

このアプリは韓国語学習者を対象にした最初の実装であり、この課題解決のモデルケースとして位置づけています。

## 今後の展望

**このアプリ（v2）の完成まで（推定 5〜7 週間）**

Phase 0〜4 を順に進め、「記事を読む → 単語を覚える → 日記でアウトプット → AI に校閲してもらう」サイクルをアプリ内で完結させます。

**プラットフォーム化（黒字化後）**

韓国語アプリで確立したモデルを英語・料理・筋トレなど他ジャンルへ横展開し、「続けたいのに続けられない大人向けの学習プラットフォーム」として育てていく予定です。

## セットアップ（.envの内容は現時点のものです。追加、更新前提）

**バックエンド・DB**

```bash
docker compose up db backend -d
docker compose exec backend bin/rails db:create db:migrate
```

**フロントエンド**

```bash
cd frontend
pnpm install
pnpm dev
```

`frontend/.env` に以下を設定してください：

```
VITE_API_BASE_URL=http://localhost:3000
VITE_UNSPLASH_ACCESS_KEY=your_unsplash_access_key
```
